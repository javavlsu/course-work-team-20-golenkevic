# Нагрузочное тестирование

JMeter план для проверки производительности маркетплейса.

## Сценарий

- **100 виртуальных пользователей**, ramp-up 30 сек, длительность 120 сек.
- Каждый пользователь по кругу запрашивает: главную, каталог, каталог с фильтром по бренду, страницу брендов, карточку товара.
- Между запросами случайная пауза 0.5–1.5 сек (имитация чтения).

## Запуск

```
jmeter -n -t load-tests/marketplace.jmx -l load-tests/results/run.jtl -e -o load-tests/results/report
```

Откройте `load-tests/results/report/index.html` чтобы посмотреть HTML-отчёт.

## Метрики, на которые смотреть

| Метрика           | Целевое значение |
|-------------------|------------------|
| 95-й перцентиль   | < 500 мс         |
| Среднее           | < 200 мс         |
| Error rate        | < 0.1%           |
| Throughput        | > 100 req/s      |

## Оптимизации, применённые после первого прогона

1. Добавлены индексы в таблицу `products` (status, category, brand, seller, price).
2. Включён Spring Cache для `categories` и `brands` (грузятся раз и держатся в памяти).
3. Пагинация на каталоге — по 20 товаров на страницу.
